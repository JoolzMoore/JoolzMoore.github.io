Grid design, responsive sites and semantic structure

Date: 3 April Updated: 26 May 2016

What is grid-based design?
A grid system is a structure comprising a series of horizontal and vertical lines used to arrange content. Grid-based design sees the screen as made up as a grid that you can move objects around in and is so much more flexible than the old table frames. The grid system provides a system to present and structure content and imagery in a manageable way. It is a recommended way to organise modules and sections of code in the front end, and also behind the scenes for websites. A good grid system can help maintain good working experience for developers, as well as, a good user experience for customers and clients using the site. It uses heirachical elements, to make information flow down in such a way that the site is easy to work with and that users find it easy to navigate, read and understand.

Whats all the hype about responsive webpages?
A responsive website responds to the environment where is viewed via media queries. The website needs to be responsive to many devices such as touchscreen smartphones, tablets, laptops and large wide-screens. Many devices have different ways of displaying type, images and color through various screen resolutions. To make information responsive it needs to be constructed so all content, images and structure will automatically size to fit to different screen widths and resolution. The re-formatted and adapted interface needs to deliver relevant and focussed user experiences in many different ways. For example, navigation on a small touchscreen device is a very different experience to using a keyboard and or mouse. Often the buttons for touchscreen need to be larger and more widely spaced out than those for mouse clicking, you canâ€™t really hover with touch-screen. So responsive sites often need to tailor the user-interaction to the device they are being viewed on. At the same time it is ideal to make sure all websites display information that is consistent and similar in layout on any device, so a customer or visitor does not get confused.

What is semantic structure?
s Semantic structure is a way of organising a language so that it represents a meaning. Semantics are in spoken languages, as well as the programming languages we use in web development. When you see a web page, you can often tell by the design which part of the page is the header (often with the navigation area) and which is the footer (with the copyright and other details). But internet bots, readers, and other applications that run automated tasks (scripts) need to interpret web pages over the internet and to 'see' what is in the design. So it is necessary to structure the elements in a meaningful and consistent way using tags and language expressions that are converted to instructions or "operational semantics" to describe the execution of the program. HTML Tags use defined classes or ID styles for the main sections of a site with names like "header" and "footer" for the DOm to operate. The code methodically describes the content, rather than how that content should look in order to identify the role and importance of each element on the page in a systematic way. Semantics is WHAT we mean the program should do, as apposed to syntax which is the language-specific constraint on how we express the semantics.
