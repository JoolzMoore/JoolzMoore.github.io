Scope


Date: 9 July

What is 'scope' and how does it work in JavaScript?
Scope relates to where a variable is placed, it is also called variable scope. The location where you declare a variable will affect where it can be used in the code. If a variable is declared within a function, it can only be used within that function. If declared within the function with a var keyword, it is called a local variable or funcation-level scope. It cannot be accessed outside of the function in which it was declared. The interpreter from the browser creates local variables when the function is run and removes them as soon as the function has finished it's task. This means that if the function runs twice, the variable can have different values each time. Local variables are only remembered during the period of time that a function is being executed.

If you create a variable outside of a function, then it can be used anywhere within the script. It is called a global variable and has global scope. Global variables are stored in the memory as long as the page is loaded into the web browser. This means that they take up more memory than local variables, and it also increases the risk of naming conflicts. If you forget to declare a variable using the var keyword, the variable will work but it will be treated as a global variable (this is considered bad practice though).
